Exiting the Shell

Why do I need to exit

If you want to close your shell you would need to use the exit command or the escape command, using ctrl+c will work but it returns and error, and ctrl+c does not work in scripts. So in Scripts you would need to use exit or escape.

Unlike ctrl+c, exit and escape return with success.

Exit

In the Shell

To exit the shell you would need to use the exit command or the escape command (or ctrl+c)

To use the command, write (in the shell) this command:

exit

This should be your output:

{} -> exit
CLOSING
if this isn't your output, you probably have an error.
To get help make an issue or find your error
message in the error list.

In a Script

To learn about scripts, go to the scripts page in the documentation.

Using the exit command in a script will close the krabb shell after executing the script. It will not break the script's process. To end the script while it is running, use the escape command documented in the Escape section.

To use the exit command in a script add to it this:

# ...
exit
# ...

After you finish your script you can run it with the src command documented it the Scripts page.

Escape

In the Shell

Escape is similar to the exit command and will exit the shell when executed (another alternative is ctrl+c but ctrl+c returns an error)

To use it in the shell you need to run this command:

escape

This should be your output:

{} -> exit
CLOSING
if this isn't your output, you probably have an error.
To get help make an issue or find your error
message in the error list.

In a Script

Unlinke the exit command, using the escape command will end your script and will not execute any code after it was called.

If you would like to use escape in a script you would need to write it like this:

# ...
escape
# ... this code would not execute ...
